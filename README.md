# ECG Signal Imputation using MIT-BIH Arrhythmia Database

A structured machine learning project exploring multiple approaches to ECG signal imputation, evaluated on the MIT-BIH Arrhythmia Database. The project progresses through five phases: data preprocessing, baseline models, a custom SAITS-style transformer, ablation studies, and a final written report.

## Project Overview

### Phase 1 — Data Preprocessing

All 48 records from the MIT-BIH Arrhythmia Database are downloaded and preprocessed through the following pipeline:

- Bandpass filtering (0.5–40 Hz) with a 50 Hz notch filter applied using `filtfilt` for zero-phase distortion
- Kurtosis-based window quality filtering to discard excessively noisy segments
- Patient-level train/val/test splitting to prevent any data leakage across splits
- Z-score normalisation applied per patient using statistics computed on the training set only
- 2x data augmentation via amplitude jitter, Gaussian noise, time warping, and baseline shift
- Masking convention of 1 = observed and 0 = missing enforced consistently throughout all phases

### Phase 2 — Baseline Models

Nine imputation methods are implemented and compared, ranging from classical interpolation to deep recurrent models. All methods are evaluated on missing positions only using the `block_target_peaks` masking strategy at 25% block ratio.

| Method | MAE | RMSE | PRD (%) | QRS MAE | Flat MAE |
|---|---|---|---|---|---|
| Transformer (Ours) | 0.4824 | 0.8432 | 43.45 | 0.9824 | 0.3682 |
| BiGRU | 0.5073 | 0.8859 | 44.30 | 1.2610 | 0.4050 |
| BiLSTM | 0.5256 | 0.8928 | 44.64 | 1.2801 | 0.4232 |
| PCHIP Interpolation | 0.7111 | 1.3590 | 68.08 | 3.3648 | 0.3512 |
| Linear Interpolation | 0.7246 | 1.3786 | 68.93 | 3.4080 | 0.3607 |
| Savitzky-Golay | 0.7248 | 1.3786 | 68.93 | 3.4082 | 0.3609 |
| BRITS | 0.7261 | 1.3579 | 67.90 | 3.4013 | 0.3631 |
| Mean Imputation | 0.7951 | 1.3534 | 67.67 | 3.3570 | 0.4477 |
| Cubic Spline | 0.8150 | 1.4376 | 72.01 | 3.2232 | 0.4884 |
| LOCF | 0.8246 | 1.3865 | 69.32 | 3.2556 | 0.4949 |

The QRS MAE and Flat MAE columns show performance separately on high-amplitude QRS regions (|z| > 1.5) versus flat isoelectric baseline regions. The large gap between QRS MAE and Flat MAE for interpolation methods highlights that classical approaches fail precisely where ECG morphology matters most clinically. BiLSTM used early stopping at epoch 35 and BiGRU at epoch 30. BRITS was implemented via PyPOTS with the version pinned to `pypots==0.4.0` due to a CUDA kernel compatibility issue on sm_60 (P100) GPUs triggered by a Kaggle Docker image update.

### Phase 3 — SAITS Transformer

A custom two-stage transformer inspired by SAITS (Self-Attention-based Imputation for Time Series) with the following architectural and training improvements:

- Multi-scale patch embedding using parallel 1D convolutions with kernel sizes 4, 8, and 16, capturing fine QRS detail, P/T wave structure, and coarse baseline trends simultaneously
- Pre-norm transformer blocks with stochastic depth regularisation to reduce overfitting on deeper stacks
- Joint ORT (observed reconstruction task) and MIT (missing imputation task) loss weighted by lambda_ort = 0.1
- Amplitude-weighted MAE loss with a factor of 3.0 on samples where absolute amplitude exceeds 1.5 standard deviations, to emphasise clinically important QRS peaks during training
- Curriculum masking schedule progressing from easy random 15% masking through medium block 20% masking to hard block-target-peaks 25% masking over the course of training
- Warmup cosine learning rate scheduler with 10 warmup epochs and a minimum learning rate of 1e-6

The model was trained with AdamW (lr=3e-4, weight_decay=1e-4) for up to 150 epochs with early stopping at patience=25. Training stopped at epoch 105 with the best checkpoint at epoch 100.

Final result: MAE 0.4824, RMSE 0.8432, PRD 43.45%, a 4.9% improvement over the best baseline (BiGRU MAE 0.5073). Bootstrap 95% CI on test MAE: [0.4123, 0.4504]. The transformer achieves the largest absolute gain on QRS regions (QRS MAE 0.9824 vs BiGRU 1.2610), which are the clinically most important parts of the signal.

### Phase 4 — Ablation Studies

Five ablations examine the contribution of individual design choices to final performance:

| Ablation | Variable | Finding |
|---|---|---|
| Masking ratio | Random 10%–50% | MAE increases monotonically with masking difficulty |
| Block size | 10%–40% | Performance degrades smoothly above 25% |
| Model depth | 1–4 transformer layers | 4 layers optimal; diminishing returns above that |
| Amplitude weight | 1.0–8.0 | Bowl-shaped curve with minimum at 3.0 |
| Patch size | Single vs multi-scale | multi (4+8+16) outperforms all single-scale variants |

Full ablation results are in the `results/` folder as individual CSV files.

### Phase 5 — Report

Full written report covering introduction, related work, methodology, experimental results, ablation analysis, and conclusions. Available in the `report/` folder.

## Running the Notebook

The notebook is self-contained and runs top to bottom. Phase 1 downloads the MIT-BIH data automatically if it is not already present. Each phase saves its outputs to `/kaggle/working/` subdirectories which the subsequent phase loads from.

The notebook was developed and executed entirely within Kaggle notebook environments using T4 x2 GPU accelerators. Because of Kaggle's weekly GPU quota limits and per-session time constraints, training was spread across multiple sessions. Where a training run had already completed, the training cell was commented out and replaced with a hardcoded result cell containing the original outputs, to allow downstream analysis and plotting cells to run without repeating expensive computation. The SAITS transformer training alone accumulated approximately 28 hours of GPU time across sessions. All hardcoded values are the exact outputs from the original training runs and have not been modified.

## Reproducibility Notes

- All random seeds are fixed at `SEED = 42` for masking, augmentation, splitting, and model initialisation
- Patient-level splitting guarantees no patient window appears in more than one of train, val, or test
- The masking convention of 1=observed, 0=missing is enforced identically across all models and evaluation functions
- The `block_target_peaks` masking strategy deliberately targets R-peak regions to simulate clinically realistic artifact patterns, making evaluation harder and more meaningful than random dropout masking
- Bootstrap 95% confidence intervals on the transformer MAE are computed over 1000 resamples of the test set

---

## Key Design Decisions

The `block_target_peaks` masking strategy was chosen for the primary evaluation condition because random dropout masking systematically underestimates real-world difficulty. Actual ECG corruption from motion artifacts, electrode detachment, and signal clipping tends to affect contiguous segments that often coincide with high-amplitude QRS complexes, which are the most diagnostically important regions of the waveform. Targeting these regions during training and evaluation creates a more realistic and challenging benchmark.

The amplitude-weighted loss was introduced because standard MSE and MAE losses are dominated by flat isoelectric baseline regions, which constitute the majority of an ECG window by time but carry little clinical information. Upweighting high-amplitude positions by a factor of 3.0 during training redirects gradient signal toward QRS reconstruction. This is reflected in the results: the transformer's QRS MAE (0.9824) represents a 22% improvement over BiGRU's QRS MAE (1.2610), a larger relative gain than on the flat baseline regions.

BRITS underperformed the simpler BiLSTM and BiGRU baselines. This is partly attributable to the PyPOTS version constraint required for CUDA compatibility, and partly to BRITS being designed for multivariate imputation across feature channels rather than univariate within-channel signal reconstruction, which is the task here.
